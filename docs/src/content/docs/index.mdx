## Why NorthstarDB?

Modern AI coding systems are moving from "one assistant in one editor" to **orchestrated swarms**:

- 100s of parallel tasks (analysis, refactor, test gen, security scans)
- Long-running sessions (hours/days)
- Distributed workers (cloud + local, multiple devs)
- Huge working sets (whole repos, dependency graphs, traces, embeddings)
- Need for replayable decisions ("why did agent X do this?")

Existing databases each miss important pieces. NorthstarDB unifies the essential primitives in one coherent system.

## Project Principles

1. **Benchmarks are law** â€” Performance changes only count if the benchmark suite agrees
2. **Correctness first** â€” Property tests + crash tests run from day one
3. **State is derived; the log is truth** â€” Data structures are rebuildable from the commit stream
4. **Pay coordination at commit** â€” Snapshot reads are always cheap
5. **Make performance visible** â€” Every subsystem ships with counters and microbenchmarks
6. **Predictable > clever** â€” Tail latency matters more than peak throughput
7. **No hidden costs** â€” Track allocations/op and bytes moved in all hot paths
