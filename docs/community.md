NorthstarDB is a community-driven project. We welcome contributions of all kinds!

### Quick Links

- **GitHub**: https://github.com/anthropics/northstardb
- **Discord**: https://discord.gg/northstardb (Coming soon)

#### Code
- Fix bugs or implement features

### Getting Help

When asking for help:
1. **Search first** - Check issues, discussions, and docs
2. **Be specific** - Include code, errors, and what you've tried
3. **Use the right channel** - Issues for bugs, Discussions for questions
4. **Be patient** - This is a volunteer community

### Reporting Bugs

Use the [bug report template](https://github.com/anthropics/northstardb/issues/new?template=bug_report.md) and include:
- NorthstarDB version
- Operating system and Zig version
- Minimal reproduction code
- Expected vs actual behavior
- Backtrace if applicable

## Code of Conduct

### Our Pledge

We pledge to make participation in our community a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity, expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

### Our Standards

**Positive behavior includes**:
- Being respectful and inclusive
- Gracefully accepting constructive criticism
- Focusing on what is best for the community
- Showing empathy towards other community members

**Unacceptable behavior includes**:
- Harassment, trolling, or derogatory comments
- Personal or political attacks
- Public or private harassment
- Publishing private information without permission
- Unprofessional conduct

### Enforcement

**Reporting**: Email conduct@northstardb.org or DM any maintainer

**Consequences**:
- Warning for first offense
- Temporary ban for repeated offenses
- Permanent ban for severe violations

**Contact**: All conduct reports are handled by the maintainer team within 48 hours.

### Attribution

This Code of Conduct is adapted from the [Contributor Covenant](https://www.contributor-covenant.org/), version 1.4.

## Communication Guidelines

### In Issues and PRs

- **Stay on topic**: One issue per topic
- **Be concise**: Respect others' time
- **Use emojis** for clarity (e.g., :bug:, :rocket:, :thinking:)
- **Link related work**: Reference issues/PRs

### In Discord

- **Use the right channel**: `#general`, `#help`, `#contributors`, `#random`
- **No DMs unless invited**: Keep discussions public
- **Time zones**: Be mindful of global community
- **Assume good intent**: Text lacks tone, give benefit of doubt
